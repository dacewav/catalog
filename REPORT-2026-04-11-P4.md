### 2. ✅ Longer Premium Animations
**Problem:** Animations for modal/wishlist open/close were too short, didn't feel premium.
**Fixes:**
- Modal overlay: `fadeInUp` .3s → .45s with `cubic-bezier(.16,1,.3,1)`

- Files: `store-styles.css`

### 3. ✅ Ctrl+Z Undo for Sliders
**Problem:** Ctrl+Z blocked on range sliders (no browser-native undo for sliders, so nothing happened).
**Fixes:**
- Differentiated `inTextInput` (input:not([type=range]), textarea) from `inRangeInput`
- Text inputs → browser-native Ctrl+Z (standard text undo)
- Range sliders, selects, non-inputs → theme undo via `undo()` / `redo()`
- Added Ctrl+Y (Windows redo convention)
- Files: `src/admin/cmd-palette.js`

### 4. ✅ Presets Apply Complete Styles
**Problem:** Selecting a preset only changed fonts, not the full color palette (surface2, text, muted, hint, borders all stayed from previous theme).
**Fixes:**
- All 12 presets now include: `bg, surface, surface2, accent, text, muted, hint, border, border2, red, redL, fontDisplay, fontBody, glowColor`
- Each preset has a fully coherent dark/light color scheme
- `applyPreset()` now calls `broadcastThemeNow()` to force iframe sync
- Files: `src/admin/core.js`

### 5. ✅ License Editor Dark Theme
**Problem:** License editing inputs rendered as white boxes, breaking dark aesthetic.
**Fixes:**
- Added `.lic-ed-row input` CSS: dark background (`rgba(12,10,14,0.7)`), proper border, focus glow
- Added hover border tint on `.lic-ed-row`
- Added `::placeholder` styling
- Files: `admin.html`

### 6. ✅ TOP Badge Position
**Problem:** "TOP" badge (top:10px right:10px) collided with wishlist heart button (top:8px right:8px).
**Fix:** Moved badge to `left:10px` instead of `right:10px`.
- Files: `store-styles.css`

### 7. ✅ Modal Player Unified with Global Player
**Problem:** Modal player operated independently, causing audio conflicts when switching between modal and global playback.
**Fixes:**
- Added `AP.playModalBeat(beat)` method — plays via global player with modal tracking
- Modal now sets `_modalMode = true` and syncs all UI (player bar, name, meta, thumbnail)
- Same-beat detection: if already playing the same beat, just resumes instead of recreating Audio
- `playModalBeat()` in cards.js now delegates to `AP.playModalBeat()`
- Files: `src/player.js`, `src/cards.js`

### 8. ✅ Per-Beat Glow Animations
**Problem:** No way to make specific beats stand out with animated effects.
**Fixes:**
- New `glowConfig` data structure per beat: `{ enabled, type, color, speed }`
- 5 glow types in admin: Pulso suave (active), Ciclo RGB (rgb), Latido (pulse), Respiración (breathe), Neón parpadeo (neon)
- Admin editor: toggle + select + color picker + speed slider in beat editor
- Store CSS: 5 animation keyframes with CSS custom properties for color (RGB components via `--glow-r/g/b`)
- Auto-disabled when beat is playing to avoid box-shadow conflicts with `.is-playing` state
- Files: `src/cards.js`, `src/admin/beats.js`, `admin.html`, `store-styles.css`
