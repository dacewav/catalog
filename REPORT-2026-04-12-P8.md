## ✅ Fix: Play button ya no reinicia animaciones

**Problema:** `_updateIcons()` en `player.js` llamaba `window.renderAll()` en cada play/pause, reconstruyendo todo el DOM → todas las CSS animations se reiniciaban desde 0%.

**Solución:** Reemplazado `renderAll()` por `_updatePlayingState()` que:
- Usa `data-id` para identificar la tarjeta correcta (no índice DOM)
- Toggle de `is-playing` class sin re-render
- Actualiza iconos play/pause directamente via DOM
- Actualiza waveform bars (`wbar.anim`) solo en la tarjeta que reproduce
- Feedback visual con `play-pulse` ring animation (no interrumpe la animación principal)

**Archivos:** `src/player.js`

**Store render:** `cards.js` emite `--anim-int` si no es 100%

## ✅ Performance hover para muchas tarjetas

- JS-driven sibling blur con throttle de 80ms (reemplaza `:has()` puro para grids grandes)
- CSS usa `.hovering` + `.hover-active` classes en vez de depender solo de `:has()`
- `will-change: transform, filter, opacity` en cards con hover effects
- El fallback CSS `:has()` se mantiene para cuando JS no ha inicializado
