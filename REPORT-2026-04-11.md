# DACEWAV.STORE — Session Report

**Date:** 2026-04-11 04:00–04:33 GMT+8

## Issues Found & Fixed

### 1. 🔴 CRITICAL: CSP blocking Firebase connection
**Symptom:** Page never loaded. Loader stayed forever. No data from Firebase.
**Root cause:** `Content-Security-Policy` `script-src` didn't include `https://*.firebaseio.com`. Firebase Realtime Database uses long-polling via dynamically injected `<script>` tags from `*.firebaseio.com/.lp?...`. CSP blocked every attempt → Firebase retried in infinite loop → nothing loaded.
**Fix:** Added `https://*.firebaseio.com` and `https://static.cloudflareinsights.com` to `script-src` and `connect-src` in `_headers` for both store and admin pages.

### 2. Hero section completely rewritten
**Problem:** 3 files fought for control of hero HTML — `index.html` hardcoded text, `settings.js` rebuilt it without styles, `theme.js` overwrote styles after settings.js. Result: glow/stroke never worked, colors conflicted.
**Fix:**
- `index.html`: Hero starts with default text (shows immediately, no JS needed)
- `settings.js`: New `_buildHero()` is the **sole owner** of hero HTML construction
- `theme.js`: No longer touches hero HTML, only section-level CSS vars (gradient, padding, eyebrow)
- CSS: `.glow-word.stroke-mode` class, `color:inherit` on `::after`
- Sync: `applySettings()` called after every `applyTheme()` to rebuild hero

### 3. Glow color mixed with text color
**Problem:** The "Color principal" picker controlled text color, glow color, AND stroke color simultaneously.
**Fix:** Three independent controls:
| Control | ID | Affects |
|---|---|---|
| Color texto | `h-text-clr` | All text including last line |
| Color del glow | `h-glow-clr` | Only text-shadow glow effect |
| Color outline | `h-stroke-clr` | Only stroke border |

### 4. Page load speed — caching
**Problem:** No data was cached. Loader blocked page until Firebase returned beats/settings/theme.
**Fix:**
- Theme, settings, and beats all cached in `localStorage`
- On repeat visits, page renders instantly from cache
- Firebase updates cache in background

### 5. Loader timeout
**Problem:** Loader required ALL 3 flags (theme + settings + beats) to hide. 5s timeout.
**Fix:** Loader hides when theme loads (not all 3). Timeout reduced to 1s.

### 6. `defer` broke page
**Attempted:** Added `defer` to `store-app.js` to unblock render.
**Problem:** `defer` caused script to execute after `load` event → `window.addEventListener('load', ...)` never fired → nothing initialized.
**Fix:** Removed `defer`. Fonts loaded async with `media="print" onload` trick instead.

### 7. Stale root-level build files
**Problem:** `store-app.js` and `admin-app.js` at repo root were old duplicates of `dist/` versions.
**Fix:** Removed from repo. Only `dist/` versions are the builds.

## Architecture Notes

### Hero rendering flow
1. HTML has default text → visible immediately
2. `applyTheme()` → sets CSS vars (gradient, padding, eyebrow styles)
3. `applySettings()` → `_buildHero()` → rebuilds hero HTML with correct glow/stroke/text colors
4. Firebase live data → overwrites cache → re-triggers `applyTheme()` + `applySettings()`

### Key rule
- **Never hardcode hero text in `index.html` beyond the default fallback**
- `_buildHero()` in `settings.js` is the ONLY function that builds hero HTML
- `theme.js` must NOT query or modify hero title elements
