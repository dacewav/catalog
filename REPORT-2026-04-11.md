# Report — 2026-04-11

## Sesión: Admin Panel Fixes

### Bugs críticos fixeados

#### 1. CSS stroke hardcodeado en hero
- **Archivo:** `admin.html` línea 440, `store-styles.css`
- **Problema:** `.glow-word` tenía `-webkit-text-stroke: 1px var(--acc)` hardcodeado. El pseudo-element `::after` tenía `opacity: 0.35` por defecto.
- **Fix:** Removido stroke hardcodeado del CSS. Default de `--hw-op` cambiado a `0`. Solo se aplican efectos cuando el usuario activa glow/stroke.

#### 2. Hero no personalizable
- **Archivos:** `src/settings.js`, `src/theme.js`, `src/admin/core.js`, `admin.html`
- **Problema:** Última línea SIEMPRE envuelta en `<span class="glow-word">` con blur encima. Solo se podía tener texto relleno con Shift+Enter.
- **Fix:** `glow-word` solo se aplica cuando `heroStrokeOn` o `heroGlowOn` están activos. Nuevo picker "Color texto" (`h-text-clr`) para color de relleno. `--hw-blur: 0` y `--hw-op: 0` cuando no hay efectos.

#### 3. Preview iframe no funcionaba
- **Archivo:** `_headers`
- **Problema:** `X-Frame-Options: DENY` bloqueaba que `index.html` se cargara en el iframe del admin.
- **Fix:** Removido `X-Frame-Options: DENY`. Agregado `frame-ancestors 'self'` al CSP.

#### 4. Admin no conectado a la tienda
- **Archivos:** `src/admin/core.js` (postMessage), `src/main.js` (message handler)
- **Problema:** postMessage origin checking muy estricto. Store rechazaba mensajes del admin.
- **Fix:** Origin checking más permisivo. Fallback a `'*'` en postMessage. Acepta `https://dacewav.store` y `null`.

#### 5. Init order — color editor no cargaba
- **Archivo:** `src/admin/firebase-init.js`
- **Problema:** `loadThemeUI()` se ejecutaba antes de `buildColorEditor()` por dynamic imports. Inputs no existían cuando se cargaban valores.
- **Fix:** `_checkReady()` espera `_uiBuilt` flag antes de llamar `loadThemeUI()`. `buildColorEditor()` corre antes.

#### 6. Cambios no se guardaban (R2 config)
- **Archivo:** `src/admin/core.js` (`_collectSiteSettings`)
- **Problema:** `_collectSiteSettings()` no incluía campos R2. Solo se guardaban con botón explícito.
- **Fix:** R2 config se auto-guarda con otros settings.

#### 7. Clases CSS faltantes para elementos JS
- **Archivo:** `admin.html` (CSS inline)
- **Problema:** 28 clases CSS que el JS genera dinámicamente no existían en el nuevo CSS.
- **Fix:** Agregadas: `.beat-row`, `.lic-ed-row`, `.inline-modal`, `.toast.err`, `body.light`, etc.
