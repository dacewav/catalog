## 🐛 Bugs Fixed

### 8. Slider Instability (partículas y glow locos)
**Causa raíz:** Triple problema:
1. `broadcastTheme()` sin debounce → decenas de broadcasts/segundo
2. Iframe ejecutaba `applyTheme()+applySettings()` en CADA mensaje
3. `initParticles()` recreaba todo desde cero en cada theme-update

**Fix aplicado:**
- `broadcastTheme()` debounced 80ms + dedup por JSON
- Iframe message handler debounced 60ms (batch messages → 1 render)
- Change detection en `initParticles()`, `applyAnim()`, `_buildHero()`, divider rebuild
- Partículas reutilizan existentes cuando solo cambia velocidad

**Archivos:** `src/admin/core.js`, `src/main.js`, `src/theme.js`, `src/settings.js`, `src/effects.js`

### 9. Undo/Redo No Funcionaba
**Causa raíz:** 3 bugs:
1. `_lastSavedTheme` nunca se actualizaba → stack no crecía
2. `_undoDebounce` no se seteaba → clearTimeout no funcionaba
3. Estado inicial no se guardaba → primer undo fallaba

**Fix:** `pushUndo()` actualiza `_lastSavedTheme` y `_undoDebounce`. `pushUndoInitial()` guarda estado al cargar.

**Archivos:** `src/admin/core.js`

### Mejoras

- [ ] **Text colorizer: sync bidireccional** — El textarea raw y el colorizer visual no se sincronizan en tiempo real. Si editas el textarea, el colorizer no se actualiza automáticamente.
- [ ] **Hero colorizer: newlines** — El colorizer del hero no maneja bien los saltos de línea del textarea. Funciona pero el flujo no es pulido.
- [ ] **Divider: más efectos** — Falta gradient background, stroke/outline similar al hero.
- [ ] **Animaciones: preview** — Sería útil ver un preview de cada animación en el dropdown del admin.
- [ ] **Glow presets: preview** — Los botones de presets no muestran qué glow van a aplicar.
- [ ] **Partículas en admin** — El preview de partículas del admin (canvas en sección hero) no se actualiza al mover sliders. Solo se inicia al entrar a la sección.
