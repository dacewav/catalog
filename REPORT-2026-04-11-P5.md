## 🔴 BUG NO RESUELTO: Círculo rojo gigante en admin preview

**Prioridad máxima para la siguiente sesión.**

### Descripción
Al editar un beat en el panel admin y configurar animaciones combo (type + type2), aparece un círculo rojo/grande en el preview `.bcpv` del editor. El bug se reportó originalmente como visible en el index del store, pero el problema real está en el **preview del admin**.

### Reproducción
- Browser: Brave (Chromium-based)
- Trigger: Editar un beat → sección Animación → seleccionar animación primaria (ej: "rebotar") + secundaria (ej: "holograma") → el preview muestra un círculo rojo semitransparente
- Ocurre con ciertas combinaciones (rebotar + holograma sí, brillo + algo no necesariamente)

### Intentos de fix (NO FUNCIONARON)
1. **CSS store-styles.css** — Eliminé reglas `anim2-holograma::after`, `anim2-brillo::after`, `anim2-cambio-color::after`, `anim2-neon-flicker::after` del selector base y de las reglas individuales. Agregué `background:transparent;border:none;outline:none` al base `::after`.
2. **CSS dist-store-styles.css** — Mismo fix que store-styles.css.
3. **JS cards.js** — Agregué filtro `_anim2FilterTypes` para evitar que animaciones basadas en filtro se apliquen como `type2`.
4. **JS beats.js** — Mismo filtro en `_applyCardStyleToPreview()`.
5. **Bundle rebuild** — Rebuild de `dist/store-app.js` y `dist/admin-app.js` con los cambios.
6. **admin.html inline CSS** — Eliminé reglas `.bcpv.anim2-holograma::after`, `.bcpv.anim2-brillo::after`, etc. del CSS inline del admin. Agregué `background:transparent` al bloque base.

### Lo que se sabe
- El `::after` pseudo-elemento del `.bcpv` tiene `content:'';position:absolute;inset:0;border-radius:inherit;z-index:1`
- NO tiene background explícito (debería ser transparente)
- El `.bcpv` tiene `overflow:visible` (igual que `.beat-card`)
- El admin NO carga `store-styles.css` — tiene sus propios estilos inline en `admin.html`
- El admin carga `dist/admin-app.js` como único script
- Los filtros CSS (`hue-rotate`, `brightness`) en el parent `.bcpv` crean un nuevo stacking context

### Hipótesis pendientes (no investigadas)
1. **Bug de Chromium/Brave** — `filter` + `position:absolute` pseudo-element + `border-radius:inherit` puede causar artefactos de renderizado en Blink
2. **`mix-blend-mode` interaction** — El `.bcpv-inner::after` tiene `mix-blend-mode:overlay` que podría interactuar mal con los filtros del parent
3. **CSS custom properties** — `--card-tint` con gradient + `mix-blend-mode:overlay` en un contexto de stacking con filtro podría producir colores inesperados
4. **Animación `rebotar` específica** — La animación `translateY` podría mover el `::after` fuera del `border-radius` clipping, exponiendo el rectángulo completo del pseudo-elemento

### Próximos pasos sugeridos
1. Abrir admin en Brave y usar DevTools para inspeccionar qué elemento DOM es el círculo
2. Probar `overflow:hidden` en `.bcpv` (cuidado con clipping del TOP badge)
3. Probar `isolation:isolate` en `.bcpv` para forzar stacking context aislado
4. Probar agregar `will-change:transform` al `::after` para forzar GPU layer
5. Si es bug de browser, agregar workaround con `transform:translateZ(0)` al `::after`
