## Deep Analysis — Hallazgos críticos

### 1. Preview Panel — Posición
- `.split` usa `grid-template-columns: 80px 1fr auto` — estructura correcta
- El preview panel es el 3er hijo del grid, debería ir a la derecha
- **Posible causa**: CSS en media queries o collapsed state puede estar rompiendo el layout
- Líneas clave: admin-styles.css:204, 774-834

### 2. Beat Save — ¿Por qué falla?
- `saveBeat()` en `beats.js` importa `db` desde `state.js`
- `db` se setea en `firebase-init.js` via `setDb(database)` Y `window._db = database`
- El `db` importado como `let` en state.js debería ser accesible via closure
- **Riesgo**: si `db` es null al momento de guardar → silent fail
- `saveBeat()` hace `db.ref('beats/' + id).set(beat)` — sin try/catch alrededor
- **Posible causa**: `db` no está inicializado cuando el usuario intenta guardar

### 3. Live Preview — Pipeline
- `_sendLiveUpdate()` envía postMessage al iframe + escribe a Firebase `liveEdits/<id>`
- Store escucha `liveEdits` node en `main.js` → `applyLiveUpdate()` → `renderAll()`
- ACK via `beat-update-ack` postMessage de vuelta al admin
- **Problema**: `beat-live-preview.js` usa `window._db` directamente en vez del módulo `db`
- Si `window._db` no está set → live edit solo funciona via postMessage (sin persistencia)

### 4. card-style-engine.js
- Módulo compartido entre admin y store — buena arquitectura
- `buildCardStyle(prefix, includeAnimSub)` construye desde DOM
- `populateCardStyle(cs, prefix)` llena DOM desde objeto
- `applyStyleToPreview(pv, cs)` aplica a elemento preview
- `mergeCardStyles(global, beat, custom)` — store usa esto
- **Potencial issue**: `includeAnimSub` param — global card style no pasa `true`, beat editor sí

### 5. Global Card Style (g- prefix)
- admin.html tiene sección `sec-card-global` con IDs `g-` prefix
- Pero NO hay código que construya cardStyle desde `g-` prefix
- `card-global.js` debería tener `initGlobalCardStyle()` — referenced from nav.js
- **Bloque crítico**: si card-global.js no funciona, el estilo global no se aplica

### 6. Hardcoded Values
- `ALL_SLIDER_IDS` en beat-presets.js — hardcoded list of all slider IDs
- `renderEffectGalleryHTML` — gallery presets hardcoded
- Holo colors default `['#ff0080', '#00ff80', '#0080ff']` hardcoded en varios sitios

### 7. Dependency Injection Pattern
- Módulos usan `setXxxDeps()` para romper circular imports
- Wiring central en `core.js` — correcto pero frágil
- Si un módulo se importa antes de que core.js wire las deps → crash silencioso

### 8. admin-main.js — Import Order
- Orden de imports puede causar problemas si un módulo depende de otro que no se ha cargado
- firebase-init.js se importa ÚLTIMO — correcto, bootstraps everything
- Pero click-handler registration happens in beats.js which runs before firebase-init

## Fix 3 — Resize handler duplicado (03:49 GMT+8)

### Problema
- `resize.js` Y `preview-resize.js` ambos manejaban el resize del preview panel
- Ambos self-initializaban, ambos escuchaban eventos de drag
- `preview-resize.js` overrideaba `window.toggleFullscreenPreview` de `resize.js`
- Doble handler causaba comportamiento errático en el resize

- `resize.js`: strippeado a solo utilidades de viewport sync (`syncPanel`, `updateViewportBtns`)
- `preview-resize.js`: single handler para resize, collapse, fullscreen, keyboard shortcuts
- `preview-resize.js` ahora llama `_syncPanel` durante resize para mantener viewport buttons sincronizados
- Bundle: -1KB (242.7 → 241.7KB)
