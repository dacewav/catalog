# 2026-04-13 — Sesión 1 (Login fix + Deploy debugging)

## Problema principal
Google login del admin panel completamente roto — el botón no hacía nada al dar click.

## Root cause
`src/admin/theme-presets.js` tenía un `Object.assign(window, { pushUndo, ... })` huérfano del refactor de core.js. `pushUndo` no se importaba en ese archivo → ReferenceError crasheaba TODO el bundle → `window.doGoogleLogin` nunca se definía → onclick del botón fallaba silenciosamente.

## Fix aplicado
- `src/admin/theme-presets.js` — eliminado el primer bloque `Object.assign(window)` de 22 líneas que referenciaba variables no importadas (pushUndo, autoSave, broadcastTheme, etc.). El segundo bloque (funciones propias del archivo) se mantuvo.

## Problemas de deploy (4 horas de debugging)

### Vercel
- Todos los deployments fallaban — Vercel no tenía `vercel.json` ni configuración de build
- Se agregó `vercel.json` con `buildCommand`, `outputDirectory`, y CSP headers
- Se movió `esbuild` de devDependencies a dependencies para Vercel
- CDN de Vercel cacheaba el HTML viejo agresivamente — no había forma de purgar con token limitado
- Se desconectó Vercel del repo al final

### Cloudflare Pages
- Deployment fallaba por submodule huérfano `catalog/` en el repo
- Causa: yo cloné el repo dentro de sí mismo en una sesión anterior
- Fix: `git rm --cached catalog` + eliminar directorio
- Después del fix, deployment exitoso ✅

## Lecciones aprendidas
- NUNCA clonar un repo dentro de sí mismo — crea submodules huérfanos
- Vercel sin `vercel.json` no ejecuta builds ni tiene output directory configurado
- La API de Vercel con tokens `limited: true` no puede acceder a deployments ni purgar cache
- Cloudflare Pages falla silenciosamente si hay submodules sin `.gitmodules`
- Verificar el build log de CI ANTES de asumir que el deploy funciona
