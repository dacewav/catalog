## Beat Image Gallery Fix (BEATS.JS)

**Problema:** Al subir imagen nueva en el editor de beats, la anterior se sobrescribía sin rastro. El usuario quería que ambas se muestren lado a lado y se guarden en galería.

**Fix aplicado en `catalog/src/admin/beats.js`:**
- `prevImg()` reescrito: ahora renderiza todas las imágenes como miniaturas gallery en `#img-prev`
- `uploadBeatImg()`: guarda URL anterior en galería antes de sobrescribir
- `saveBeat()`: auto-guarda todas las imágenes de `_beatImgHistory` a galería
- Variables globales: `_selectBeatImg`, `_removeBeatImg`, `_clearBeatImgHistory`
- `_beatImgHistory` se resetea en `openEditor()` para beat nuevo y edición

**⚠️ NO VERIFICADO EN NAVEGADOR.** El usuario reportó que no funcionó. Necesita debugging real.

## BUG: Paneles de tabs se despliegan en otros paneles

- CSS: `.etp { display: none !important; }` / `.etp.on { display: block !important; }`

- Hay ~15 secciones `.section` con misma lógica display none/block
