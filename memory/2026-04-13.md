## Beat Image Gallery Fix (BEATS.JS)

**Problema:** Al subir imagen nueva en el editor de beats, la anterior se sobrescribía sin rastro. El usuario quería que ambas se muestren lado a lado y se guarden en galería.

**Fix aplicado en `catalog/src/admin/beats.js`:**
- `prevImg()` reescrito: ahora renderiza todas las imágenes como miniaturas gallery en `#img-prev`
- `uploadBeatImg()`: guarda URL anterior en galería antes de sobrescribir
- `saveBeat()`: auto-guarda todas las imágenes de `_beatImgHistory` a galería
- Variables globales: `_selectBeatImg`, `_removeBeatImg`, `_clearBeatImgHistory`
- `_beatImgHistory` se resetea en `openEditor()` para beat nuevo y edición

**⚠️ NO VERIFICADO EN NAVEGADOR.** El usuario reportó que no funcionó. Necesita debugging real.

### 2. No revisar dependencias cruzadas
- Cambiar `prevImg()` puede romper cualquier cosa que llame esa función
- `prevImg()` es llamada desde: `oninput` del campo `f-img`, `uploadBeatImg`, `openEditor`, `_selectHistoryImg`, gallery picker callbacks, `_initPvImgUpload`
- **REGLA: Antes de reescribir una función, grep exhaustivo de TODOS sus callers**

- Asumí que `#img-prev` era el único contenedor de imagen
- Hay DOS secciones de imagen: `#img-prev` en la tab media Y `#pv-img-preview` en la tab extras
- El `pv-img-upload` handler en beat-preview.js tiene su propio flujo que podría chocar
- **REGLA: Mapear TODOS los elementos DOM relacionados antes de tocar lógica**

### 4. No verificar que el HTML generado se parsea bien
- El HTML generado con `innerHTML` con onclick inline puede tener problemas de escaping
- Las comillas dentro de strings de template pueden romper el render
- **REGLA: Después de generar HTML dinámico, verificar que los onclick handlers funcionan**
